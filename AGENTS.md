# Инструкции для агентов проекта

## Язык общения

- Всегда отвечай пользователю на русском языке.

## Запросы на коммит

Когда пользователь пишет `коммит` или иначе просит сделать коммит:

1. Запусти проверку, которая соответствует области изменений.
2. Сформируй сообщение коммита в формате `<текущая-ветка>. <Описание изменения с большой буквы>`.
   - Пример: `main. Обновить структуру UI компонентов`.
   - Название ветки получи командой `git branch --show-current`.
3. Добавляй в коммит только релевантные файлы.
4. Сразу после успешного коммита выполни push.
5. Если push не прошел из-за авторизации, сети или отсутствующего upstream, сообщи точную причину и оставь локальный коммит на месте.
